# 🐞 [Bug Report] 컨트롤러 조작 중 네트워크 전환 시 스킬 입력 지연 및 캐릭터 프리징 (예시)

| 이슈 번호 | 심각도 | 우선순위 | 플랫폼 | 상태 |
| :--- | :---: | :---: | :--- | :--- |
| **DNF-001** | **Critical** | **P1** | AOS (S24), Controller | Open |

## 1. 개요 (Summary)
* 블루투스 컨트롤러를 사용하여 스위프트 마스터 플레이 중, Wi-Fi에서 LTE로 네트워크가 전환될 때 약 2~3초간 조작 입력이 무시되며 캐릭터가 프리징되는 현상.

## 2. 테스트 환경 (Environment)
* **기기:** Galaxy S24 (Android 14)
* **네트워크:** Wi-Fi 6 <-> LTE (Handover)
* **조작 도구:** Xbox Wireless Controller (Bluetooth 연결)
* **버전:** v1.0.5 (남마법사 업데이트 패치)

## 3. 재현 단계 (Steps to Reproduce)
1. 블루투스 컨트롤러를 S24 기기에 연결 후 게임 진입.
2. 스위프트 마스터 캐릭터로 시나리오 던전 입장.
3. '소닉 무브' 스킬을 사용하여 고속 이동 중인 상태 유지.
4. 기기 상단 바를 통해 Wi-Fi를 강제로 해제하여 LTE로 네트워크 전환(Handover) 유도.
5. 컨트롤러의 스킬 버튼 및 조이스틱을 연속 입력하며 캐릭터 상태 확인.

## 4. 기대 결과 (Expected Result)
* 네트워크 전환 시 짧은 로딩 혹은 지연이 발생하더라도, 컨트롤러의 입력 데이터는 유실 없이 서버로 전송되어 스킬이 발동되어야 함.

## 5. 실제 결과 (Actual Result)
* 네트워크 전환 직후 약 3초간 컨트롤러의 모든 입력이 무시됨.
* 캐릭터는 이동 모션에서 멈춘 채 프리징되며, 3초 후 이전 입력값이 한꺼번에 반영되거나 위치 롤백(Rubber Banding) 발생.

## 6. 증거 자료 (Evidence)

## 7. 분석 의견 (QA Note)
* 터치 조작 시에는 지연 시간이 1초 내외로 짧으나, 컨트롤러 사용 시 입력 버퍼가 초기화되는 현상이 관찰됨. 
* 네트워크 재접속 로직과 블루투스 입력 모듈 간의 충돌 가능성 확인 필요.
