# [분석] 남마법사 업데이트 기능 분석 및 리스크 식별

## 1. 신규 클래스 메커니즘 분석
### 1.1. 남마법사 공통 로직: '불사'
- **기획 의도:** 사망 시 즉시 부활하여 일정 시간 생존 후 HP를 회복하는 생존 특화 로직.
- **QA 관점 리스크:** - 특정 상태 이상(빙결, 석화) 중 사망 시 로직 미발동 가능성.
  - 부활 직후 무적 판정과 몬스터의 공격 판정 간의 프레임 경합(Race Condition).

### 1.2. 스위프트 마스터: '기동성 및 범위 타격'
- **기획 의도:** 빠른 이동과 동시에 넓은 범위를 타격하는 스피디한 액션.
- **QA 관점 리스크:** - 초고속 이동 시 클라이언트-서버 간 좌표 동기화 오차(Desync).
  - 지형지물(벽, 장애물)에 끼이거나 맵 밖으로 이탈하는 현상.

## 2. 플랫폼 환경 분석 (PC vs Mobile)
- **UI/UX:** 모바일의 좁은 화면 내 스킬 버튼 배치와 PC 키보드 매핑 간의 조작 편의성 격차 분석.
- **성능:** 남마법사의 화려한 파티클 이펙트가 저사양 기기(Android Go 등)에서 메모리 부족(OOM)을 유발할 가능성 식별.
