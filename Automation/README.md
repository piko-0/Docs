# 🤖 Automation: 테스트 자동화 설계 (Draft/Simulation)

본 폴더에 포함된 코드는 실제 테스트 자동화 환경 구축 및 효율화를 위한 **개념 증명(PoC) 및 설계 예시**입니다.

## 1. 작성 배경 및 목적
- **업무 효율화:** 라이브 서버의 반복적인 리소스 부하 테스트를 수동으로 진행할 시 발생하는 인적 리소스 낭비를 방지하고자 함.
- **설계 역량 증명:** Appium 및 Python을 활용하여 실제 게임 환경에서 어떻게 자동화 로직을 구성할 수 있는지에 대한 설계 프로세스를 기술함.

## 2. 자동화 시나리오 예시
- **남마법사 스킬 무한 루프:** 신규 클래스의 화려한 이펙트 중첩 시 발생할 수 있는 메모리 누수(Memory Leak)를 식별하기 위해 500회 이상 스킬 사용 반복 수행.
- **튜토리얼 자동화:** 신규 캐릭터 생성 시마다 반복되는 초반 진입 구간을 자동화하여 실시간 검증 시간을 단축함.

> **[Note]** > 본 스크립트는 실제 구동 환경이 아닌 **설계 및 로직 예시**로 작성되었습니다. 
> 향후 Appium 서버 및 기기 연결 환경이 구축된 상태에서 즉시 커스터마이징하여 적용 가능한 수준의 코드로 구성되어 있습니다.
